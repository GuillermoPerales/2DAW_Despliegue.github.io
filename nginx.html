Practica 4

Documentar correctamente el proceso de instalación y configuración de un servidor Nginx que dispone de dos sitios web:

  • Primer Sitio
    • Creas el fichero de configuracion asociado dentro de site-avaliable, para ello copiamos el archivo default

      cd /etc/nginx/sites-avaliables
      sudo cp default miSitio



    • Cambiamos el fichero de configuracion para poner que se escuche en el puerto 82 y la direccion sea /var/www/miSitio

      sudo nano miSitio



    • Creamos el directorio con el contenido de nuestro sitio y lo habilitamos haciendo manualmente un enlace simbolico de la carpeta sites-enabled con sites-avaliable


      sudo ln -s /etc/nginx/sites-available/miSitio /etc/nginx/sites-enabled



    • Situamos los log en la carpeta /etc/logs/miSitio
      Para ello creamos la carpeta y añadimos la directiva access_logs en el archivo de configuracion.

    • Activar la compresion de los archivos que superen los 80k, para ello usaremos el modulo de HttpGzip que es un módulo de Nginx que se encarga de la compresión y descompresión "al vuelo" de ficheros.
      Primero habilitaremos la directiva en el archivo de configuracion principal


    • Dispone de una página que se mostrará al acceder a una ruta que no exista. Esta página (404.html) estará situada en /var/www/errores
  • Segundo Sitio
    • Creamos la aplicacion Node y la arrancams en el puerto 3001


      • Creamos el sitio con los siguientes prerrequisitos:
      • Estar publicado en el puerto 81
      • El directorio donde se encuentra el contenido del sitio será /var/www/sitioNode
      • Los logs se sitúan en el directorio /etc/logs/sitioNode
      • Dispone de una página que se mostrará al acceder a una ruta que no exista. Esta página (404.html) estará situada en /var/www/errores
      • Al acceder a http://localhost:81/documentación se producirá una redirección a la página oficial de nodejs (https://nodejs.org/en/)
    • Creamos el directorio donde estara el contenido del sitio y le otorgamos permisos


    • Creamos el archivo de configuracion copiando el default de Nginx

      cd /etc/nginx/sites-available
      sudo cp default sitioNginxNode

    • Y cambiamos las directivas necesarias para nuestra configuracion


    • Creamos el enlace simbolico a mano

      sudo ln -s /etc/nginx/sites-available/sitioNginxNode /etc/nginx/sites-enabled

    • Para situar los log en una carpeta concreta añadimos la directiva access_log a la configuracion


    • Para añadir una pagina que mostrar cuando se accede a un aruta que no existe, primero creamos la carpeta errores en /var/www/errores
      alli creamos un archivo html con la pagina del error
      Luego añadiremos la directiva error_page 404 a nuestra configuracion


      Reiniciamos servidor y lo compraobamos


    • Para acceder al sito de Nodejs al entrar en /documentacion tenemos que añadir la directiva rewrite a la configuracion del sitio
