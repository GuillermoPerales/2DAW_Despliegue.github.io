Git-Flow

En esta practica, vamos a simular el sistema de ramificaciones propuesto por Vincent Driessen (Gitflow)

  • Crear un repositorio con las ramas de Master y develop usando Git-flow
    • Instala Git-flow

      $sudo apt-get install git-flow

    • Inicializa un repositori de Git vacio y establece la configuracion de las ramas

      $git flow init



    • Creamos el repositorio online y hacemos el primer commit




      $git add fichero_prncipal.txt
      $git commit -m "Add fichero principal"
      $git push --set-upstream origin develop

  • Generar diferentes features con pequeños cambios en el proyecto. Algunas de las features deben modificar un mismo fichero
    • Creamos las features con sus archivos, modificamos algunos y las subimos

      $git flow feature start feature_01



      $git checkout feature/feature_01



      $git add
      $git commit -m
      $git push

      $git flow feature start feature_02





      $git add
      $git commit -m
      $git push

    • Ahora finalizamos las features

      $git flow feature finish feature_01




      $git flow feature finish feature_02






      $git push

  • Creamos una nueva release con todas las features creadas.

    • $git flow release start release_01




    • Hacemos el push de la release

      $git add ."
      $git commit -m "Subida release"
      $git push

    • Creamos el tag de la version y hacemos el push

      $git tag -a NombreTag -m "Comentario"
      $git push --tag

  • Simular la existencia de un error en producción y solucionarlo por medio de un hotfix.
    • Creamos una rama de hotfix

      $git flow hotfix start hotfix01




    • Modificamos y solucionamos los errores
    • Finalizamos el hotfix

      $git flow hotfix finish hotfix01
